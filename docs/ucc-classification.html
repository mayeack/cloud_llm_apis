Round-trip result (vs live deployed tree, 2026-07-20)

Residual diffs, all accepted:

  1. VERSION — trailing-newline only. The doubled 1.1.0 line is ucc-gen's own convention (version + build-version), not a bug; the prior plan's "fix VERSION" hygiene item is void.
  2. default/app.conf / appserver/templates/base.html — per-build epoch stamp.
  3. appserver/static/js/build/globalConfig.json + openapi.json — intended fixes: version 1.0.1 → 1.1.0 (live built copy was stale) and activities interval default.
  4. bin/import_declare_test.py — live copy was hand-tweaked (quote style, dirname(__file__) path insert). Generated form accepted: Splunk puts the script's own bin/ on sys.path, so package imports work either way.
  5. bin/TA-*_rh_compliance_activities.py — live had hand-edited default '60' (string); source standardizes on 300 matching the deployed spec and the configured input.
  6. appserver/static/customfavicon/ — empty directory in live tree; cruft, not reproduced.

Recovered-drift decisions

  • compliance_activities interval default: live tree disagreed with itself (spec said 300, built globalConfig + rh said 60). Source standardizes 300.
  • metadata/local.meta and local/** are instance state: excluded from git entirely.
